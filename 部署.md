# Next.js 部署完整教程

> 目标：把本地跑的 Next.js 项目，部署到一台 Linux 服务器上，用户能通过域名访问。

---

## 前置准备

- 一台 Linux 服务器（Ubuntu 22.04 推荐）
- 一个域名（可选，没有用 IP 也行）
- 本地代码可以正常 `npm run dev`

---

## 第一步：服务器安装 Node.js

```bash
# 安装 nvm（Node 版本管理工具）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 重新加载环境变量
source ~/.bashrc

# 安装 Node.js 20
nvm install 20
nvm use 20

# 验证
node -v   # v20.x.x
npm -v    # 10.x.x
```

---

## 第二步：安装 PM2（进程守护）

PM2 的作用：让 Next.js 在后台持续运行，服务器重启后自动拉起。

```bash
npm install -g pm2
```

---

## 第三步：把代码传到服务器

**方式一：Git（推荐）**

```bash
# 服务器上
git clone https://github.com/你的用户名/你的项目.git /home/ubuntu/myapp
cd /home/ubuntu/myapp
```

**方式二：直接 scp 上传**

```bash
# 本地执行，把项目压缩上传
scp -r ./myapp ubuntu@你的服务器IP:/home/ubuntu/
```

---

## 第四步：配置环境变量

```bash
cd /home/ubuntu/myapp

# 创建生产环境变量文件
cp .env.example .env.production
vim .env.production
```

把本地 `.env` 里的内容填进去，比如：

```
DATABASE_URL=postgresql://user:password@localhost:5432/mydb
JWT_SECRET=你的密钥
```

> ⚠️ 注意：`.env.production` 不要提交到 Git！

---

## 第五步：安装依赖 & 打包

```bash
cd /home/ubuntu/myapp

# 安装依赖
npm install

# 打包（这一步编译前后端所有代码）
npm run build
```

打包成功会看到：

```
✓ Compiled successfully
✓ Collecting page data
✓ Generating static pages

Route (app)                    Size
├ ○ /                          1.2 kB
├ ○ /pages/user/login          3.4 kB
└ ƒ /pages/talks/list          2.1 kB
```

---

## 第六步：用 PM2 启动服务

```bash
# 启动
pm2 start npm --name "myapp" -- start

# 查看运行状态
pm2 status

# 查看日志
pm2 logs myapp

# 设置开机自启
pm2 startup
pm2 save
```

这时候访问 `http://你的服务器IP:3000` 应该能看到网站了。

---

## 第七步：安装 Nginx 做反向代理

> 为什么需要 Nginx？
>
> - 处理 HTTPS
> - 静态文件缓存加速
> - 80/443 端口转发到 3000

```bash
# 安装 Nginx
sudo apt update
sudo apt install nginx -y

# 启动
sudo systemctl start nginx
sudo systemctl enable nginx
```

创建配置文件：

```bash
sudo vim /etc/nginx/sites-available/myapp
```

填入以下内容（先用 HTTP，后面再加 HTTPS）：

```nginx
server {
    listen 80;
    server_name 你的域名.com;  # 没有域名就填服务器IP

    # 静态文件直接由 Nginx 返回，不经过 Node.js，更快
    location /_next/static/ {
        alias /home/ubuntu/myapp/.next/static/;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 其他请求转发给 Next.js
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

启用配置：

```bash
# 软链接到 sites-enabled
sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/

# 测试配置有没有语法错误
sudo nginx -t

# 重载配置
sudo systemctl reload nginx
```

现在访问 `http://你的域名.com` 就能看到网站了。

---

## 第八步：配置 HTTPS（可选但强烈推荐）

```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx -y

# 自动申请证书并配置 Nginx
sudo certbot --nginx -d 你的域名.com

# 证书自动续期（certbot 会自动创建定时任务）
sudo certbot renew --dry-run
```

完成后 Nginx 配置会自动更新，支持 HTTPS，访问 `https://你的域名.com` 搞定。

## 日常更新代码流程

```bash
cd /home/ubuntu/myapp

# 拉最新代码
git pull

# 重新安装依赖（如果 package.json 有变化）
npm install

# 重新打包
npm run build

# 平滑重启，不中断用户
pm2 reload myapp
```

## 常用命令速查

```bash
# 查看 Next.js 运行日志
pm2 logs myapp

# 重启服务
pm2 restart myapp

# 停止服务
pm2 stop myapp

# 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log

# 查看 Nginx 访问日志
sudo tail -f /var/log/nginx/access.log
```

## 整体架构图

```
用户浏览器
    ↓ HTTPS 443
Nginx
    ├── /_next/static/* → 直接返回静态文件（快）
    └── 其他请求 → 转发到 localhost:3000
                        ↓
                    Next.js (PM2守护)
                        ↓
                    数据库
```

## 常见问题

**Q：打包报错怎么办？**

先在本地跑 `npm run build`，确保本地没错再上服务器。

**Q：启动后访问 3000 端口没反应？**

检查服务器安全组/防火墙是否开放了 3000 端口，或者直接用 Nginx 做代理就不需要暴露 3000。

**Q：更新代码后页面没变化？**

确认执行了 `npm run build` 然后 `pm2 reload`，缺一不可。

**Q：数据库连不上？**

检查 `.env.production` 里 `DATABASE_URL` 是否正确，服务器能否访问数据库地址。
